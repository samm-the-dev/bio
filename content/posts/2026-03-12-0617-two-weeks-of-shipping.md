Jeez, I did a lot in the last couple of weeks. Developing these side projects with Claude has become a strange addiction. I think I'm getting a lot out of it though, both in experience with developing with Claude and useful/interesting applications. I didn't realize that the entirety of two of my projects happened in the last couple of weeks. Both feel decently mature to me now, though I'll admit I still need to be better about examining the code output.

Here's a technical summary of each project's changes generated by Claude, followed by some thoughts and context added by me.

## [Build-a-Jam](https://build-a-jam.app)

> Build-a-Jam got PWA support and a custom domain (build-a-jam.app) in early March, then a round of UX polish in mid-March: copy-as-custom for exercises, hide/show toggles, swipe-to-dismiss, wake lock to keep the screen on during practice, deep link sharing for exercises and sessions, and a cleaned-up queue layout and picker dialog.

_This was my original side project, started to help me learn React. I'll admit I haven't truly learned React all that well, instead I became more obsessed with organically developing an AI-assisted dev workflow with Claude. I need to do a deeper technical dive on both React and Claude to actually grok them, but it seems like things are working well enough so far._

_Anyway, back to this app specifically, I created this to possibly help me run improv jams with my peers without necessarily relying on a more experienced director. I have some ambitions to grow my improv skills and being able to run jams on my own would help that. This app hasn't been tested much but these changes were motivated by some feedback I got from one of my improv mentors, plus lessons I learned developing my other apps._

## [Toolbox](https://github.com/samm-the-dev/toolbox)

> The shared toolbox repo (templates, AI context files, shared utilities) got renamed from `planet-smars` to `toolbox` and reorganized during the first week of March. Additions since Feb 27: Google Drive sync libraries and GCP auth infrastructure, a GitHub Pages deployment guide, a reusable Claude Code GitHub Actions workflow (on-demand @claude mentions, adversarial auto-review, and Copilot response modes), and a share utility with Web Share API and clipboard fallback.

_This was something I wanted to develop very early on in my work with Claude. I knew I'd be spinning up more projects, and I knew I'd want to share resources between them, so I decided to try out an approach where my shared utilities live in a central toolbox repo which is consumed via Git submodule. That approach has its limitations and friction points, which I'm sure I'll write about in more detail later, but overall has worked out well enough._

_Early on this had a React+Vite template, but I've since split that out into its own template repo (which includes this repo as a submodule). The additions you see here are all things I picked up while working on my React apps. The Claude Code GHA workflow is brand new, shipping in the same PR as this blog, so I haven't started working with it yet._

## [Ohm](https://apps.samm-the.dev/ohm)

> Ohm was created on Feb 27 and went from zero to a fully featured ADHD kanban app in two weeks — 26 PRs total.
>
> The first week built the foundation: shadcn/ui primitives, PWA with Google Drive sync, index-based data model with a capacity system, persistent GCP Cloud Function for Drive auth, Tailwind v4, and a custom domain.
>
> The second week (March 10–12) was a feature sprint: energy budget system with rolling time windows, gradient meters, and configurable caps; settings redesign; soft delete for card instances; activity scheduling with Drive sync; day groups in columns with energy subtotals; a budget bar fixed to the bottom of the screen; and a day focus dialog for cross-column scheduling views.

_I've been working on my mental and physical health recently, and as a result my energy has been higher which seems to have made my ADHD worse in some ways. This app was built to help me manage that, and to extend the utility I've gotten out of using my Google calendar a lot more recently. I started with the tech stack I used for Build-a-Jam, and discovered a lot of new things while developing this app. PWA, Google integration, and several UI patterns were all new things I learned while figuring out what I wanted this to be._

_I still have more I want to do with it, including some big ideas for an ecosystem of task apps, but it's in a solid usable state now. Feel free to try it out and give me feedback._

## This site

> Bio launched March 4 and went from scaffold to feature-complete in about eight days. The content stack went through a detour: started with local YAML, migrated to Sanity CMS, then migrated back to YAML + markdown within a day. The final setup — `js-yaml`, `gray-matter`, `marked`, and `shiki` generating TypeScript data files at build time — requires no dashboard and no network at build time.
>
> Features that landed across 31 PRs: markdown blog pipeline with syntax highlighting and RSS, Bluesky cross-posting, a shows page with calendar downloads and map links, a GIF collection served from Google Cloud Storage CDN, tabbed feeds pulling in Bluesky and Letterboxd activity, project deep linking with share buttons, search and tag filtering across pages, automated accessibility auditing in CI, and daily scheduled rebuilds to keep Open Graph metadata fresh.

_The creation of this site was motivated by a few different desires, but ultimately prompted by my experience trying out a couple of voice acting classes the weekend prior. I don't see myself fully switching careers into that industry, but it's something I'd like to pursue as a hobby at least, and I've noticed it demands a more rich online presence. This prompted a whole quiet rebrand of all of my social accounts, which are now linked on the home page here._

_The content management journey was interesting. I set out with the intention for only code/infrastructure to be AI-generated, with content and prose being generated by me, so I wanted a clean separation between the two. I started with a JSON file that piped through a script, decided to try out a CMS service for a better editing experience, but ultimately decided that editing markdown in VS Code suited me just fine (that's what I'm doing now!)_

_All in all this site is an effort to put myself out there even more than I have been for the last year-and-change of doing improv, which I've found very rewarding. It also gave me a place to host the many (200+) GIFs I've made, a place to host upcoming improv show info, and more opportunities to improve my web dev skills._

I'm proud of what I've gotten done in the last couple of weeks, but I'm feeling some of my old screen life bad habits creep back in, so I'm gonna make an effort to develop apps less and touch grass more for the next couple of weeks.
